# スキルマップ／組織ダッシュボード 画面設計書

格納先：`docs/dashboard-design.md`
関連ファイル：`dashboard_mockup.html`（画面モック）

本書はresumeプロジェクトの新規画面「スキルマップ／組織ダッシュボード」の設計書であり、Claude Codeへの実装指示を兼ねる。モックHTMLはUI・挙動・集計ロジックの仕様書を兼ねており（JS内の関数がロジック定義そのもの）、モック内のデータはハードコードのサンプルなので、実装では既存DBからの集計に置き換える。本書とモックに齟齬を見つけた場合は実装前に確認すること。

## 背景・目的（重要度順）

1. メンバーが資格を取得したら各自が登録することで、別途実施していた資格取得アンケートを廃止し、リアルタイムに集計へ反映する（登録機能は既存の資格登録画面を利用。本画面は集計結果の表示のみ）
2. 各組織のスキル・資格保有状況を視覚的に把握できるようにする（組織的な目標設定に役立てる）
3. 周囲の社員のスキル・資格を把握し、次に何を取るかのヒントにする（個人的な目標設定に役立てる）

## 前提

- resumeプロジェクトの新規画面として実装する（Next.js App Router / TypeScript / Prisma / Auth.js v5 / Neon PostgreSQL / Tailwind CSS v4）
- 集計はサーバーサイドで行う（Server Component または Route Handler + Prisma）。Auth.jsのセッションからログインユーザーを特定し、「💡おすすめ」判定に使用する
- 既存のスキル・資格マスタ、社員、組織、保有情報のテーブルを利用する。**新規テーブルは追加しない**
- **閲覧権限**：本画面のスキル・資格の保有者名は**全社公開**とする。経歴書本体の「同一部署スコープ」ルールとは別ルールなので、経歴書閲覧の権限チェックを流用しないこと
- モックのCSSはデザイン指定の参考値。実装はTailwind CSS v4で、モックと同等の見た目・配色・余白感を再現する

## 画面構成

### ヘッダー

- タイトル＋右側にKPI「資格保有 ◯件 ▲ +◯件（前年度末比）」
- 年度の定義は **3月1日〜翌2月末**。前年度末比＝今年度内の新規取得件数

### 🎉ティッカー（お祝い表示）

- 今年度に取得された資格を、取得日の新しい順に**すべて**ローテーション表示（4〜5秒間隔のフェード切替、ホバーで一時停止）
- 文言：「◯/◯ ◯◯さん（部署名）が 資格名 を取得しました！」。氏名と資格名を強調色
- 取得日が**直近3か月以内**の項目にはNEWバッジ
- 右端に「n / 全◯件（今年度）」のカウンタ

### 大タブ：［資格］［スキル］［ヒートマップ］

スクロールを最小化するため、コンテンツはすべてタブ切替で表示する。

### 共通フィルタ

#### 部署タブ（単一選択）※資格タブ・スキルタブ

- 選択肢は「全社」＋**事業部・部の単位**（`organization_unit` の3階層のうち上位2階層まで。表示名の例：SI／クラウド／金融／流通／DX／ERP）。**グループ（第3階層）は表示しない**
- 実際の部署名・並び順は組織マスタから動的に取得する
- 部を選択した場合、その配下グループ所属者を含めて部単位にロールアップして集計する
- タブが表示幅に収まらない場合は、タブ行に**横スクロール**を付ける（折り返しはしない）
- ※モックの「サンプルA〜C」は横スクロールと空状態（データ0件の部署）の確認用で、実装対象ではない

#### カテゴリチップ（複数選択・デフォルト全ON）※資格タブ・スキルタブ・ヒートマップタブ

- 資格＝国家資格／ベンダー資格／その他 等、スキル＝プログラミング言語／クラウド・インフラ／データベース／業務知識 等。**カテゴリの実体は既存マスタに合わせること**（マスタ追加でカテゴリが増える前提の作りにする）
- チップは選択中✓表示、**最後の1つはOFF不可**（クリック無効）
- 選択していないカテゴリは集計から除外する
- チップが表示幅に収まらない場合は横スクロール

### 資格タブ・スキルタブ

#### ドーナツチャート＋凡例（構成比）

- 選択中カテゴリの合算に対する資格別（スキル別）シェア。**上位5項目＋「その他（◯種）」**に集約
- 凡例行クリックで保有者名をチップ形式で展開（全社表示時は「氏名（部署名）」、部署選択時は氏名のみ）
- 「その他」行クリックで残り項目を「名称（◯名）」のチップで展開
- 該当0件の場合は「該当する登録がありません」の空状態を表示

#### 💡おすすめ（資格タブ・構成比ビューのみ）

- **判定ロジック**：ログインユーザーが保有していない資格のうち、選択中スコープ（部署 or 全社）で保有者数が多いもの（保有者2名以上・上位3件）に💡を付与
- ツールチップ：「◯◯で◯名が保有している人気の資格です」（スコープが全社なら「全社で〜」）
- **文言ルール（重要）**：「未取得」「持っていない」などのネガティブな表現をUI上に一切出さないこと。判定ロジックとしての非保有チェックは実装するが、画面文言には使わない

#### パネル下の注記（ビューごとに出し分け）

- 構成比ビュー：「💡＝あなたへのおすすめ／凡例クリックで保有者を表示」
- 年度推移ビュー：「棒グラフをクリックすると、その年度の取得者を表示します」（💡の注記は年度推移では表示しない）
- スキルタブ：「凡例クリックで保有者を表示」

#### 表示切替（資格タブのみ）：構成比｜年度推移

- 年度推移＝年度（3月始まり）ごとの新規取得件数の縦棒グラフ。範囲は「〜N年度」に古い分を畳み、直近5年度＋今年度を表示
- カテゴリを2つ以上選択中はカテゴリ別の積み上げ棒、1カテゴリのみ選択中は単色棒
- 今年度の棒を強調（枠線＋ラベル「（今年度）」）
- **縦軸スケールは固定**：目盛り線と目盛りラベルを表示する。縦軸の上限は**全社・全カテゴリでの年度別最大件数を5の倍数に切り上げた値**とし、部署変更・カテゴリ増減ではスケールを変えない（絞り込み前後の棒の高さを視覚的に比較できるようにするため）
- **取得者の表示**：棒（年度）をクリック（フォーカス）すると、グラフ下にその年度の取得一覧を表示する。内容は「取得日（月/日）・氏名・資格名」のチップで、取得日の新しい順。部署・カテゴリの絞り込みを反映し、全社表示時は氏名に部署名を併記する。同じ棒を再クリックで閉じる
- スキルタブには年度推移はない（構成比のみ）

### ヒートマップタブ

- 2つの切替：**［資格｜スキル］×［人数｜保有率］**の2×2＝4パターン
- 行＝部署（在籍人数を併記）。部署は事業部・部の単位で全部署を行に表示する
- 列＝カテゴリ。カテゴリチップの選択が列に反映される。列が多く表示幅に収まらない場合はテーブルに横スクロールを付ける
- **人数＝延べ人数**（そのカテゴリ内の保有登録件数の合計。1人が同一カテゴリ内で複数保有する場合は重複してカウントする）
- **保有率＝延べ人数 ÷ 部署在籍人数**（%表示・四捨五入）。延べ人数ベースのため**100%を超えることを許容**する。人数の少ない部署が不利にならないための表示
- 画面上に注記を記載する：「人数は延べ人数（1人が複数保有する場合は重複してカウント）。そのため保有率は100%を超える場合があります」
- セルの濃淡5段階（モックの閾値を参考に、実データの分布に合わせて調整可。保有率は100%超も想定した閾値にする）

## 実装しないもの

- 検索ボックス、集計ボタン
- 統計カード行（登録メンバー数・登録スキル種類・保有者1名スキル）
- Excel出力
- 属人化リスク一覧
- 「選択外カテゴリを『その他』にまとめる」機能（検討したが不採用）
- 資格登録フォーム（既存画面があるため。本画面は表示専用）

## データ取得・実装方針

- 取得日（資格の取得年月日）を使うため、保有情報テーブルの該当カラムを確認して使用すること
- 集計クエリはN+1を避け、必要な情報（資格・スキル・保有者・部署・取得日）をまとめて取得してからサーバーサイドで整形する
- 年度計算・NEW判定・おすすめ判定・年度別取得者一覧・延べ人数・縦軸固定は、モックJSの `fy()` / `isNew()` / `recommendMap()` / `acquisitionsIn()` / `renderHeat()` の `gross()` / `TREND_AXIS_MAX` の実装と同一のロジックにする
- 部署のロールアップ（グループ所属者を部単位に集約）は `organization_unit` の自己参照階層をたどって行う
- 取得日が未登録のデータが存在し得る場合の扱い（ティッカー・年度推移から除外し構成比には含める、等）は実装前に確認すること

## 進め方のルール

- スキーマとの対応（カテゴリの持ち方、取得日カラム、事業部・部と階層レベルの対応等）や仕様の解釈に迷う点は、**実装を進めずに必ず事前に質問**すること
- 型の不一致・構文エラーなど正解が一意な客観的誤りは、修正のうえ報告でよい

## 受け入れ条件

- [ ] ヘッダーKPIの総件数・前年度末比がDB集計と一致する（年度＝3/1〜翌2月末）
- [ ] ティッカーが今年度の取得を新しい順に全件ローテーションし、直近3か月にNEWが付く
- [ ] 部署タブが「全社＋事業部・部」の単一選択で、グループは表示されず、配下グループの所属者が部単位に集計される
- [ ] 部署タブ・カテゴリチップ・ヒートマップの列が溢れた場合に横スクロールで操作できる
- [ ] カテゴリチップが複数選択・デフォルト全ONで動作し、最後の1つはOFFできない
- [ ] ドーナツが上位5＋「その他」に集約され、凡例クリックで保有者チップが展開される（全社公開）
- [ ] 💡が「非保有×スコープ内2名以上×上位3件」で付与され、ツールチップにネガティブ文言がない
- [ ] パネル下の注記がビューごとに出し分けられ、年度推移ビューに💡の注記が表示されない
- [ ] 年度推移の縦軸スケールが部署・カテゴリの変更で変化せず、目盛り線が表示される
- [ ] 年度推移の棒をクリックすると、その年度の取得一覧（取得日・氏名・資格名、絞り込み反映、新しい順）が表示され、再クリックで閉じる
- [ ] ヒートマップが資格／スキル×人数／保有率の4パターンで切り替わり、人数が延べ人数・保有率が延べ÷在籍人数（100%超許容）で計算され、その旨の注記が画面にある
- [ ] 廃止項目（検索・統計カード・Excel出力・属人化リスク・登録フォーム）が画面に存在しない
- [ ] `scripts/verify.sh` が通る（lint / typecheck / test）
