# 021 REF008 スキルマップ／組織ダッシュボード

## 目的

組織のスキル・資格保有状況を可視化する全ロール向けダッシュボードを提供する(docs/screens.md REF008)。

## 前提(依存するplan)

- 020 REF002 経歴書一覧(保有者名からREF003への遷移可否に閲覧範囲判定を使う)

## 実装内容

- `app/(authenticated)/skill-map/page.tsx`+`lib/skill-map.ts`+`components/skill-map/`: REF008
  - 集計対象: 現職かつ`is_registered=true`の社員(退職者・論理削除除外)。年度は3月1日〜翌2月末。非選択カテゴリは集計から除外
  - ヘッダーKPI: 資格保有の延べ総件数と前年度末比(=今年度の新規取得件数)
  - 🎉ティッカー(`AcquisitionTicker.tsx`): 今年度取得の資格全件をマーキーで途切れず流す(ホバーで一時停止、カウンタ併記、直近3か月にNEWバッジ、0件時プレースホルダ)
  - 大タブ[資格][スキル][ヒートマップ]+部署タブ(全社+事業部・部。Grは親の部にロールアップ、折り返さず横スクロール)+カテゴリチップ(デフォルト全ON・最後の1つはOFF不可)
  - 構成比: ドーナツ+凡例(上位5+その他)。凡例クリックで保有者名チップ展開(全社時は部署名併記)。0件時は空状態表示。保有者名からREF003への遷移は閲覧範囲判定に従う
  - 💡おすすめ: 資格タブ・構成比のみ。未保有かつスコープ内保有者2名以上の上位3件に付与
  - 年度推移: 資格タブのみ。直近5年度+それ以前集約の縦棒(カテゴリ積み上げ・今年度強調・全社全カテゴリ基準の固定スケール)。棒クリックで取得一覧の展開/収納
  - ヒートマップ(`HeatmapPanel.tsx`): [資格|スキル]×[人数|保有率]。行=全部署(事業部・部)+在籍人数、延べ人数/延べ÷在籍(100%超許容・注記表示)、濃淡5段階。初期表示行はログインユーザーの所属事業部+配下の部。行の×非表示→下部タグで再表示(最低1行)、行ドラッグ並び替え(永続化しない)
  - 集計ロジック(`lib/skill-map.ts`)は純粋関数化し単体テストで網羅する
- 検索ボックス・集計ボタン・Excel出力等は持たない(表示専用)

## 受け入れ基準

- 集計対象・年度・カテゴリ除外の定義が仕様どおり(テストで担保)
- 3タブ・部署タブ・カテゴリチップ・構成比・年度推移・ヒートマップの表示と操作が仕様どおり動く
- EDT004で資格を登録すると本画面に反映される

## 検証方法

1. `lib/skill-map.test.ts`で年度境界(3/1・2月末)・延べカウント・ロールアップを網羅する
2. Playwrightで3タブの表示・チップ/凡例/棒クリック・ヒートマップの行操作を確認する
3. `npm run verify`が通ることを確認する
