# UI改修フィードバック一括対応(第1弾・第2弾+追加調整)

## Context

REF008 スキルマップ/組織ダッシュボード改修(`2026-07-15-09-skill-map-dashboard.md` 参照)後のユーザーレビューで、私の経歴書・経歴書一覧/詳細・アカウント管理・マスタ管理・スキルマップ・共通入力部品にまたがるUI改善フィードバックを2回に分けて受領し、一括対応した。本ドキュメントは両弾と追加調整をまとめた実施記録。

### ユーザー確認済みの決定事項

1. **経験年数はDB列を月数に変更**(`experience_years` → `experience_months`)。私の経歴書(REF004)・経歴書詳細(REF003)は「◯年◯か月」表示、一覧・検索は従来どおり年
2. **(仮登録)は表示ルールで実現**: `employee.name`には素の氏名のみを保存し、「氏名あり かつ `is_registered=false`」の間だけ表示側で「◯◯（仮登録）」を付与。本人のEDT001保存でのみ解除
3. **検索カードの開閉はトグル方式**: ヘッダの「検索後に閉じる」ON/OFFトグル(デフォルトON)と開閉操作の記憶(sessionStorage・タブ単位)のみで決定。条件有無による自動開閉は廃止
4. **現場マスタの主管部署は部(DEPARTMENT)のみ**から単一選択(任意)
5. **入社前(前職)の経験は「社外プロジェクト」現場のマスタ登録運用で対応**(社外区分カラムは追加しない)。同一社員×同一現場の複数プロジェクトは許容(docs/decisions.md参照)

## DB変更(prisma migrate)

- `20260715055330_rename_experience_years_to_months` — `employee.experience_years`(年)を`experience_months`(月)にリネーム。既存36名分をプロジェクト期間の和集合から再計算してバックフィル
- `20260715062526_add_site_organization_unit` — `site.organization_unit_id`(主管部署FK・任意)を追加

## 共通UI部品(components/ui/)

- **`ClearableInput`**(新規) — フィールド内右端に×(全消去)を持つテキスト入力。`list`属性パススルーで**サジェスト(datalist)付き入力にも適用**。classNameで`px-*`/`py-*`を渡すと既定パディングを外す(クラス競合の防止)
- **`DateField` / `MonthField`**(新規) — `min`/`max`必須付与で**年4桁制限**(4桁入力後は月へ自動移動)。ネイティブのカレンダーインジケータを非表示にし、**×(右端)の左に自前のカレンダーアイコン**(`showPicker()`)を配置。`defaultWhenEmpty`で空フォーカス時の初期値を注入
- **`CollapsibleSearchCard`**(新規) — 検索条件カードの開閉(REF002/REF007共通)。ヘッダ「▼ 検索条件」(▼は左)のすぐ右隣に「**検索後に閉じる**」トグル(デフォルトON)。`notifySearchExecuted()`をフォームの検索実行時に呼び、ONならカードを閉じる。開閉・トグルはsessionStorageに記憶
- **`LoadingOverlay`**(新規) — 検索実行(`useTransition`)中に画面全体へ表示するスピナーオーバーレイ。「検索後に閉じる」でカードがhiddenになっても消えないようカードの外に配置
- **ボタンの共通スタイル** — `globals.css`に`button:not(:disabled){cursor:pointer}`を追加し、濃色ボタン(`hover:bg-zinc-700`)・枠線ボタン(`hover:bg-zinc-50`)のホバー色を全画面へ横展開

## 画面ごとの変更

### 私の経歴書(REF004 / EDT001〜004)

- 経験年数を「◯年◯か月」表示(`formatExperienceMonths`)。0年は「◯か月」、0か月は「◯年」
- 所属Grと最寄駅の行の高さを統一(最寄駅に路線名/駅名の内側ラベル、selectはh-[42px])
- 卒業年月は空フォーカスで**学年考慮の予想値**を初期表示: 1〜3月生まれ=生年+22年の3月、4〜12月生まれ=生年+23年の3月(`lib/graduation.ts`)
- 社員ID・メールアドレスはセクション編集では非表示(初回登録画面のみ)
- 「◯◯の編集」見出しを廃止し、`EditableSection`が編集/非編集で同じ位置に見出しを表示。**編集中はヘッダに[保存][キャンセル]を横並び**(保存は`form`属性でフォームをsubmit、`setPending`で送信中状態を反映。初回登録の単独画面はフォーム下部に保存)
- 経歴概要/自己PRの登録フォームとAI生成パネルを同一の枠・高さに統一
- スキル/資格は**登録済み分をタグ表示**(×で削除、タグ押下で編集展開: スキル=習熟度のみ、資格=取得年月日/有効期限のみ)。追加は「+ スキルを追加」「+ 資格を追加」→1行コンパクトフォーム。送信は従来どおり全行hidden inputの全置換

### プロジェクト経歴編集(EDT005)

- 使用スキルを同じタグ型UIに統一(タグ押下でバージョンのみ変更可)。日付は`MonthField`、現場名サジェストは`ClearableInput`

### 経歴書一覧(REF002)

- 検索条件を開閉カード化。項目順: 氏名カナ→経験年数→所属組織→スキル→資格→携わったプロジェクト→退職者を含める→検索/クリア
- 氏名検索は氏名+カナのOR部分一致。経験年数検索は月換算(`min*12`〜`max*12+11`)
- **携わったプロジェクトはセレクトボックス化**(選択ボタン廃止、選ぶだけで条件)
- 検索実行中は`LoadingOverlay`を表示

### アカウント一覧(REF007)・登録/編集(EDT006/007)

- 検索条件を開閉カード化(項目順: 氏名カナ→所属組織→権限→状態→検索/クリア)。氏名+カナのOR部分一致。検索中オーバーレイ
- 最終ログインのソートはnull(-)を常に末尾(`nulls: "last"`)
- EDT006に氏名(任意)を追加。**社員IDは既存最大値+1(6桁ゼロ埋め)を初期表示**(`lib/employee-id.ts`、変更可)
- EDT007は初回未登録(`is_registered=false`)のときのみ氏名を編集可
- (仮登録)表示は`employeeDisplayName()`(`lib/employee-labels.ts`)に一元化

### マスタ管理(MST001/002/005)

- スキル/資格マスタ: 名称のあいまい絞り込み(部分一致・大文字小文字無視、ヒットカテゴリ自動展開)
- 現場マスタ: **主管部署**(部のみ・任意)の登録/編集/一覧表示と、現場名+主管部署での絞り込み。主管部署として参照中の組織単位は削除不可。現場名と主管部署のフィールドは同一高さ(h-8)
- 社外プロジェクトはマスタ登録運用(コード変更なし。docs/decisions.md参照)

### スキルマップ(REF008 ヒートマップ)

- 初期表示行=ログインユーザーの所属事業部配下(`defaultHeatmapVisibleIds`。未所属は全行)
- 上部の部署チップを廃止。各行の×で非表示→テーブル下部にタグとして横並び→タグ押下で再表示(最低1行維持)。ドラッグ並び替えは維持
