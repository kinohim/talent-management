# 015 master-skills スキルマスタ管理

## 目的

スキルカテゴリ・スキル・バージョンを管理職が管理できるようにする(docs/screens.md master-skills)。

## 前提(依存するplan)

- 014 master-org-units 部署マスタ管理(マスタ管理ハブ・開閉式一覧の見た目を踏襲)

## 実装内容

- `app/(authenticated)/master/skills/page.tsx`+`actions.ts`: master-skills
  - カテゴリ単位の開閉式一覧(見出しに件数表示)。初期表示はカテゴリ見出しのみ
  - 画面最上部の「カテゴリを追加」フォーム(100文字・スキルカテゴリ内で一意。重複時「既に登録されているカテゴリ名です。」)
  - カテゴリ見出し右端の追加ボタンでカテゴリ直下にスキル追加フォームを展開
  - スキル名の絞り込み(部分一致・大文字小文字無視。ヒットしないカテゴリは非表示、ヒットしたカテゴリは自動展開)
  - スキル編集ではカテゴリの変更(既存選択/新規作成)も可能
  - バージョンはタグ複数入力。保存時にバージョン1件以上で`has_version=true`、0件でfalse。`display_name`は「スキル名+半角スペース+バージョン名」で自動生成
  - バージョンタグの削除は参照有無で分岐: `employee_skill`/`project_skill`から参照されていれば`is_active=false`(非表示化)、参照なしなら物理削除(`lib/skill-master.ts`)
  - スキルの削除はconfirm-dialogで確認。参照中なら「使用中のため削除できません」
  - 同名の論理削除済みスキル・カテゴリがある場合は復活させる(docs/screens.md マスタ管理系の共通仕様)

## 受け入れ基準

- カテゴリ追加・スキル追加・編集・削除・絞り込みが仕様どおり動く
- `has_version`/`display_name`の自動設定と、バージョン削除の分岐(非表示化/物理削除)が機能する
- 参照中スキルの削除がブロックされ、削除済み同名の再登録が復活として動く

## 検証方法

1. `lib/skill-master.test.ts`で参照判定・削除分岐を網羅する
2. Playwrightでカテゴリ→スキル→バージョンの追加・編集・削除の一連を確認する
3. `npm run verify`が通ることを確認する
