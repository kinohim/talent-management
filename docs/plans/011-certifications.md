# 011 資格登録

## 目的

保有資格(カテゴリ・資格名・取得年月日・有効期限)をmypage[表紙]タブのセクションとして登録・編集できるようにする(docs/screens.md mypageの資格セクション)。

## 前提(依存するplan)

- 008 mypage 私の経歴書(セクション骨格)
- 004 シードデータ(資格マスタ)

## 実装内容

- 資格セクション(`components/certifications/`): スキルセクションと同じタグ型UI
  - 登録済み資格は「取得日+資格名」のタグ表示。×で削除、タグ押下で取得年月日・有効期限のみ変更可
  - 「+ 資格を追加」でカテゴリ/資格名/取得年月日/有効期限の1行フォームを追加(認定団体は資格名選択時に行の下へ小さく自動表示)
  - カテゴリはマスタからの選択のみ。資格名はカテゴリで絞り込んだサジェスト付き入力
  - 日付は`yyyy/mm/dd`が全桁見える幅で表示
- バリデーション(`lib/certification-schema.ts`+単体テスト): 取得年月日は必須かつ**JST基準の本日以前**、有効期限は任意で取得年月日より後(同日は不可)
- 保存Server Action `saveCertifications`(`app/(authenticated)/mypage/actions.ts`に置く。007・008と同じ集約先): 本人の`employee_certification`を入力内容で置き換える(deleteMany→createManyの洗い替え)。同じ資格の再取得は新規レコードとして追加できる(重複制約は設けない)

## 受け入れ基準

- 追加・日付のみ変更・削除・保存が仕様どおり動く
- 「本日以前」判定がJST基準で行われる(UTC深夜帯でもずれない)
- 有効期限≦取得年月日はエラーになる

## 検証方法

1. `lib/certification-schema.test.ts`で日付境界(本日・未来・期限逆転)を網羅する
2. Playwrightで追加・変更・削除・保存の一連を確認する
3. `npm run verify`が通ることを確認する
