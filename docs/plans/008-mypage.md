# 008 mypage 私の経歴書(タブ・セクション編集の骨格)

## 目的

本人の経歴書の閲覧・編集を1画面`/mypage`に統合する。[表紙]/[実績]の2タブと「セクション単位で編集モードに切り替えて保存する」骨格を作り、基本情報セクション(basic-info相当)を最初のセクションとして組み込む(docs/screens.md mypage、docs/decisions.md「私の経歴書(mypage)の1画面統合」)。

## 前提(依存するplan)

- 007 basic-info 基本情報登録(フォームを共用)

## 実装内容

- `app/(authenticated)/mypage/page.tsx`+`actions.ts`: 私の経歴書。一般社員/管理職のみアクセス可(人事・営業はリダイレクト)
- `components/my-resume/MyResumeTabs.tsx`: [表紙](デフォルト)/[実績]のタブ。タブ切替はクライアントサイドの表示切替とし、編集途中の状態を保持する。タブ状態はURLクエリ`?tab=projects`で保持(`lib/my-resume-tabs.ts`)
- `components/my-resume/EditableSection.tsx`: セクション共通の編集モード切替
  - 通常は閲覧表示、「編集」でそのセクションのみフォームに切替、保存で閲覧表示に戻る(画面遷移なし)、「キャンセル」で入力破棄
  - セクション見出しは編集/非編集で同じ位置・同じ文言。編集中は見出し行の右側に[保存][キャンセル]を横並び表示し、フォーム下部には保存ボタンを置かない
- 閲覧表示はresume-detail(経歴書詳細)と同じ表示コンポーネントを使う前提の構成にする(013で共通化)
- 基本情報セクション: 007のフォームをセクションとして組み込む(社員ID・メールアドレスは表示しない)
- PDF出力の「準備中」タイル(`components/resumes/ResumeExportButtons.tsx`)をタブ下部に配置する

## 受け入れ基準

- [表紙]/[実績]タブが切り替わり、`?tab=projects`でリロードしても実績タブが復元される
- 基本情報セクションの編集→保存が画面遷移なしで完結し、保存後に最新の閲覧表示へ戻る
- キャンセルで入力が破棄される
- 人事・営業は`/mypage`にアクセスできない

## 検証方法

1. Playwrightでセクション編集の往復(編集→保存/キャンセル)とタブ復元を確認する
2. `npm run verify`が通ることを確認する
