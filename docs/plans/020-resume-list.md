# 020 resume-list 経歴書一覧

## 目的

経歴書の横断検索・一覧を提供する。人事・営業/管理職は全社員、一般社員は閲覧範囲内のみを対象とする(docs/screens.md resume-list)。

## 前提(依存するplan)

- 013 resume-detail 経歴書詳細(遷移先)
- 019 アカウント管理(検索カード・ページング・列ソート/フィルタ・復元の仕組みを共通化して流用)

## 実装内容

- `app/(authenticated)/resumes/page.tsx`+`lib/resume-search.ts`: resume-list
  - 検索対象は`is_registered=true`かつ論理削除されていない社員。退職者はデフォルト除外でフィルタ切替可能
  - 一般社員は閲覧範囲(部署一致/事業部直下は事業部一致/未所属は本人のみ)にサーバー側で自動限定し、所属組織フィルタの選択肢も範囲内のみとする(`resolveResumeViewScopeUnitIds`)。URLクエリで範囲外の組織idを直接指定しても閲覧範囲との積集合に丸める
  - 検索条件カード(account-listと同じ開閉・トグル・記憶・クリア)。項目順は氏名カナ→経験年数→所属組織→スキル条件→取得資格条件→携わったプロジェクト(現場)→退職者を含める
  - 所属組織はカスケード階層選択(初期は事業部のみ、チェックで配下展開。配下件数と説明文を表示。親子両方選択時は最深の選択配下が対象)
  - スキル・資格条件はマスタ複数選択+サジェスト+AND/ORラジオ。現場は単一セレクト(過去〜現在のいずれかのプロジェクトが一致)
  - 経験年数は`experience_months`を月換算で範囲判定(0〜99、下限≦上限は自動スワップ、上限は端数月を含む)
  - 検索結果列: 氏名/所属組織/経験年数/主なスキル/主な資格(重複除去後の上位3件+超過は「…」)。「詳細」でresume-detailへ
  - 列ソート(氏名カナ・所属組織・経験年数)+列フィルタ(氏名テキスト、所属組織は選択式で**配下を含めて判定**、経験年数範囲、スキル・資格は検索フォームと同仕様)。列フィルタは検索条件とAND合成
  - ページング(10/30(既定)/50/100)・検索実行中のローディングオーバーレイ・resume-detailからの戻りで状態復元
- `lib/resume-search.ts`のクエリ構築は単体テストで網羅する

## 受け入れ基準

- 一般社員の検索結果が閲覧範囲内に限定され、URL操作でも範囲外が漏れない
- `is_registered=false`の社員が結果に出ない
- AND/OR・現場・経験年数・退職者フィルタ・列フィルタ(所属組織の配下含む判定)が仕様どおり動く

## 検証方法

1. `lib/resume-search.test.ts`で検索条件の組み合わせ(AND/OR・範囲・閲覧範囲限定)を網羅する
2. Playwrightで3ロールの検索・詳細遷移・戻りの状態復元を確認する
3. `npm run verify`が通ることを確認する
