# 業務経歴書 Webアプリ（PWA）設計資料

コーディング着手用の資料一式。各ファイルは単体で読めるが、まず本ファイルで全体像を把握すること。

## ファイル構成

| ファイル | 内容 |
|---|---|
| `screens.md` | 画面一覧（18画面＋共通部品1）と画面ごとの入力項目・バリデーション |
| `schema.md` | テーブル定義（19テーブル、システムカラム含む） |
| `er-diagram.md` | ER図（Mermaid） |
| `screen-flow.md` | 画面遷移図（Mermaid） |
| `decisions.md` | 主要な設計判断の記録（なぜこの仕様にしたか） |
| `plans/` | 実装計画（Plan）の保存先。実装完了後、ユーザーの承認を得てファイル化する |

本資料はスキーマ・画面仕様・画面遷移の「今の正解」のみを簡潔に記述する。設計レビュー時の詳細な意思決定の経緯・画面モックアップ画像は、本リポジトリの外で別管理している人間向け資料（Excel/Word）に記録されている。本リポジトリにはそれらのファイルは含めていない。

## システム概要

社員が自分の業務経歴書をWebブラウザ（スマートフォン含む）から作成・編集し、PDF形式で出力できるPWA。PDF出力時は氏名のイニシャル化・個人情報のマスクを選択できる（社外提出用）。人事・営業／管理職は全社員の経歴書を横断検索できる。一般社員も自分の閲覧範囲内（原則同一部署。事業部直下所属者は同一事業部）の社員の経歴書を閲覧できる。

## ロール

- `一般社員`: 自分の経歴書の作成・編集・出力。加えて**閲覧範囲内（原則同一部署。事業部直下所属者は同一事業部）の社員の経歴書の閲覧**が可能（判定ルールはscreens.mdのresume-list参照）
- `人事・営業`: 全社員の経歴書の横断検索・閲覧（編集不可）。**自分の経歴書は作成しない（「私の経歴書」なし）**
- `管理職`: 人事・営業の権限 ＋ アカウント管理・マスタ管理・組織単位管理。自分の経歴書編集も可（一般社員の権限を包含）

## 技術方針の要点

- 認証: SSO（Azure AD／Google／GitHub、いずれもOIDC/OAuth）のみ。ID/パスワードによるログインは提供しない（実装済みプロバイダはGitHubのみ。詳細はscreens.mdのlogin参照）
- 退職判定: `employee.employment_status`（現職／退職）で判定する。論理削除用の`deleted_at`とは別軸
- 名前の重複: スキル名・資格名・現場ポジション名・現場名はシステム全体でユニーク（カテゴリをまたいでも重複不可）
- 出力: PDFのみ（Excel出力は提供しない）。PDF出力は未実装で、実装済み画面には「準備中」表示のみ配置している
- PWA: オフライン編集・ホーム画面追加導線・プッシュ通知は対応しない
- データ移行: 既存Excelからの移行・インポート機能は提供しない（新規入力のみ）
- 社外アクセス: 許可（Azure AD条件付きアクセスでのMFA必須化は検討中、未確定）
- ナビゲーション: 全画面パンくずリスト対応
- **言語: 利用者は日本人のみのため、画面表示・エラーメッセージ等のユーザー向け文言はすべて日本語で統一する**。Auth.js等ライブラリが提供するデフォルト画面（ログイン画面・エラー画面等）は英語表示になるため、そのまま出さず自前の日本語画面に差し替えること
