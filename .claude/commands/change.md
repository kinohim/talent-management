以下の修正を行ってください: $ARGUMENTS

必ず change-flow スキルの手順に従うこと。要点:

1. まず規模判定(仕様変更/バグ修正/軽微修正)を含む実装計画を提示する
2. 仕様が変わるなら docs/ の設計書を実装より先に修正する
3. docs/tests/ のテスト仕様書で確認観点を提示し、合意を得てからテストを実装する
4. 新テストは red(実装前に落ちる)→ green(実装後に通る)を両方確認する
5. 完了前に change-flow スキルのチェックリストを自己確認し、npm run verify を通す
